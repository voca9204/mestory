# ME.STORY - 100년 일기 앱 PRD

## 1. 제품 개요
ME.STORY는 개인의 삶을 완전히 기록하고 맥락화하는 장기 일기 앱입니다. 
특정 날짜의 "해상도"를 최대한 높여, 개인 기록과 시대적 맥락을 함께 제공합니다.

## 2. 핵심 가치 제안
- **완전한 기억 복원**: 특정 날짜의 모든 맥락 (개인+세상) 한눈에 확인
- **100년 아카이브**: 장기간 데이터 축적과 패턴 발견
- **다차원 내비게이션**: 시간축과 토픽축의 유연한 탐색

## 3. 주요 기능

### 3.1 데이터 수집 (마이크로 - 개인 데이터)
- **핵심 기록**: 일기 작성, 사진 업로드
- **운동 데이터**: Google Maps 운동 기록 연동
- **생활 기록**: 차계부, 경조금 관리
- **기념일 관리**: 생일, 결혼, 기일 등 추적
- **SNS 연동**: Facebook, Instagram, Twitter 데이터 수집
- **메신저 분석**: KakaoTalk, Telegram 사람별 필터링
- **통화 기록**: 일별 통화 목록 수집

### 3.2 맥락 데이터 (매크로 - 세상 데이터)  
- **날씨 정보**: 위치별 일별 날씨 기록
- **뉴스**: 주요 사건사고, 정치, 경제 뉴스
- **스포츠**: 주요 경기 결과, 이슈
- **문화 콘텐츠**: 음악, 영화, 드라마, 만화, 게임 관련 뉴스

### 3.3 내비게이션 시스템
- **편년체 뷰**: 수평(월일) × 수직(연도) 2D 타임라인
- **기전체 뷰**: 토픽별 히스토리 (운동, 건강, 경조사, 인간관계 등)
- **하이퍼텍스트 연결**: 관련 기록들의 비선형 탐색

### 3.4 UI/UX 요구사항
- **메인 인터페이스**: 2차원 달력 그리드 (수평: 월일, 수직: 연도)
- **상세 뷰**: 선택된 날짜의 모든 기록 레이어 표시
- **필터링**: 데이터 타입별, 사람별, 토픽별 필터
- **검색**: 전체 기록 횡단 검색
- **시각화**: 데이터 패턴과 트렌드 차트

## 4. 기술 요구사항

### 4.1 프론트엔드
- **플랫폼**: 웹 애플리케이션 (Vite + React)
- **상태관리**: Redux Toolkit 또는 Zustand
- **라우팅**: React Router
- **UI 라이브러리**: Tailwind CSS + Headless UI
- **성능**: 대용량 데이터 렌더링 최적화 (React.memo, useMemo, 가상 스크롤링)

### 4.2 백엔드
- **서비스**: Firebase 기반 BaaS
- **데이터베이스**: Firestore (실시간 동기화 + 오프라인 지원)
- **인증**: Firebase Authentication
- **스토리지**: Firebase Storage (사진, 첨부파일)
- **호스팅**: Firebase Hosting
- **Functions**: Firebase Cloud Functions (데이터 처리, 외부 API 연동)

### 4.3 외부 연동
- **Google APIs**: Maps, Calendar, Photos
- **SNS APIs**: Facebook Graph, Instagram Basic Display, Twitter API  
- **뉴스 API**: 실시간 뉴스 수집
- **날씨 API**: 정확한 일별 날씨 데이터
- **메신저 연동**: KakaoTalk, Telegram 데이터 추출

### 4.4 데이터 아키텍처
- **ETL 파이프라인**: 다양한 소스의 데이터 정규화
- **데이터 웨어하우스**: 장기 보관과 분석을 위한 구조
- **백업/복구**: 100년 데이터 보존 전략
- **프라이버시**: GDPR 준수, 데이터 소유권 보장

## 5. 개발 단계

### Phase 1: MVP (3개월)
- 기본 일기 작성/조회 기능
- 간단한 달력 내비게이션  
- 사진 업로드/관리
- 기본 검색 기능

### Phase 2: 데이터 연동 (4개월)
- SNS 데이터 수집
- 날씨/뉴스 API 연동
- 메신저 데이터 추출
- 2D 내비게이션 구현

### Phase 3: 고급 기능 (4개월)
- 토픽별 뷰 구현
- 데이터 분석/시각화
- 고급 검색/필터링
- 성능 최적화

### Phase 4: 스케일링 (3개월)
- 백업/동기화 시스템
- 다중 기기 지원  
- 데이터 내보내기
- 보안 강화

## 6. 성공 지표
- **데이터 완성도**: 일별 기록 커버리지 80% 이상
- **사용자 참여**: 월 활성 사용자 일기 작성 빈도
- **데이터 가치**: 검색/필터링 사용 패턴
- **장기 보존**: 5년+ 지속 사용자 비율

## 7. 위험 요소
- **개인정보 보호**: 민감한 데이터 취급
- **API 의존성**: 외부 서비스 정책 변경
- **데이터 볼륨**: 장기 축적 시 성능/비용 이슈
- **사용자 피로**: 데이터 입력 부담
